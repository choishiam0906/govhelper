## 1. UI/UX 개선 (Toss UI 가이드 기반)

### 1.1 UX 라이팅 개선
- [ ] **해요체 일관성 검토** - 모든 문구를 해요체로 통일
- [ ] **능동형 문장 전환** - "~됐어요" → "~했어요" 패턴 수정
- [ ] **긍정적 표현 적용** - "없어요" → "있어요" 패턴으로 에러 메시지 개선
- [ ] **캐주얼한 경어** - "~시겠어요?" 같은 과도한 경어 제거
- [ ] **CTA 버튼 명확화** - 버튼 라벨이 다음 행동을 명확히 안내하도록 수정

### 1.2 다크패턴 방지 점검
- [ ] 서비스 진입 시 바텀시트 팝업 제거 (알림 동의 포함)
- [ ] Back 버튼 시 인터럽트 없이 이전 화면으로 이동
- [ ] 모든 모달/바텀시트에 닫기 버튼 확인
- [ ] 예상치 못한 광고/팝업 제거

### 1.3 그래픽 리소스 최적화
- [ ] 문맥에 맞는 아이콘 사용 검토
- [ ] 아이콘 크기 24~40px 범위 확인
- [ ] 다크/라이트 모드 대응 확인
- [ ] 불필요한 장식적 요소 제거

### 1.4 반응형 디자인
- [ ] 모바일 기준 375px 레이아웃 점검
- [ ] 태블릿/데스크톱 반응형 테스트

## 3. 기술적 개선 (TRD 기반)

### 3.1 데이터 수집 개선
- [ ] 크롤러 스케줄러 개선 (node-cron 또는 K8s CronJob)
- [ ] 중복 데이터 방지 (URL 해시 기반)
- [ ] K-Startup 크롤링 추가 (Headless Browser)
- [ ] 기업마당 크롤링 개선

### 3.2 HWP 문서 처리
- [ ] hwp-extract 라이브러리 연동
- [ ] PyMuPDF Pro로 HWP/HWPX 파싱
- [ ] 표(Table) 데이터 마크다운 변환
- [ ] 메타데이터 자동 추출

### 3.3 RAG 검색 엔진 구축
- [ ] PostgreSQL pgvector 확장 설치
- [ ] 시맨틱 청킹 구현 (LangChain RecursiveCharacterTextSplitter)
- [ ] 텍스트 임베딩 (text-embedding-3-small)
- [ ] 하이브리드 검색 (Vector + Keyword)
- [ ] Reciprocal Rank Fusion (RRF) 구현

### 3.4 LLM 응답 스트리밍
- [ ] FastAPI SSE 엔드포인트 구현
- [ ] 실시간 토큰 스트리밍 UI
- [ ] 참조 출처 표시 기능

### 3.5 결제 시스템 강화
- [ ] Toss Payments Webhook 연동
- [ ] 카카오페이 API 연동 (tid 세션 관리)
- [ ] 네이버페이 결제창 연동
- [ ] 결제 실패 재시도 로직
- [ ] 멱등성(Idempotency) 처리

### 3.6 보안 및 인프라
- [ ] 민감 정보 AES-256 암호화 (사업자번호)
- [ ] API Rate Limiting 구현
- [ ] WAF 설정 (SQL Injection, XSS 방지)
- [ ] 분산 추적을 위한 trace_id 로깅

## 5. 작업 우선순위 요약

### P0 (즉시)
1. UX 라이팅 개선 (해요체, 긍정적 표현)
2. 다크패턴 방지 점검
3. CTA 버튼 명확화

### P1 (단기)
1. HWP 파일 다운로드 기능
2. RAG 검색 엔진 구축 (pgvector)
3. LLM 응답 스트리밍

### P2 (중기)
1. 카카오 로그인
2. 카카오페이/네이버페이 결제
3. K-Startup 크롤러 추가

### P3 (장기)
1. 엔터프라이즈 기능
2. 팀 계정
3. API 제공
